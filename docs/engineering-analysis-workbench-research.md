# 工程分析工作台调研与落地路线

更新时间：2026-06-06

## 范围边界

Railwise 工程分析工作台只服务轨道、交通、铁路、工程测量和监测场景。当前阶段不做偏离这一主线的非轨道交通测量监测类能力，也不做复杂交付流转或复杂审批闭环；这些内容不进入工具命名、样例数据、路线规划和验收标准。

工作台的目标收敛为四件事：

1. 能读入现场常见数据：JSON、CSV/TSV/分号 CSV、Excel 粘贴表、GeoJSON、LandXML、DXF、Leica GSI 和 DAT/坐标表。
2. 能完成核心计算：距离方位、角度换算、坐标转换、控制网平差、CPIII 复核、线路放样、线路里程偏距、轨道精调、盾构姿态、变形、水位、测斜和轴力。
3. 能做基本质控：限差判断、残差/RMS、速率预警、超限点、字段错误和数据缺失提示。
4. 能输出正常成果：明细表、汇总表、趋势图、Excel、Word/Markdown 报告和可追溯的计算参数。

## 调研依据

| 来源 | 调研要点 | 落地要求 |
| --- | --- | --- |
| JAG3D: https://github.com/applied-geodesy/jag3d | 大地测量最小二乘平差覆盖水准、方向、距离、GNSS 基线和变形分析。 | 控制网工具必须输出残差、RMS、权重/精度和可复核的平差摘要。 |
| QGIS Processing: https://docs.qgis.org/3.44/en/docs/user_manual/processing/intro.html | 处理工具强调参数、批处理、历史和可复现。 | 每个工具要记录输入格式、参数、处理步骤和成果表目录。 |
| QGIS Georeferencer: https://docs.qgis.org/3.44/en/docs/user_manual/managing_data_source/georeferencer.html | GCP、变换类型、残差和 RMS 是坐标转换验收核心。 | 坐标转换要支持同名点解算 2D Helmert，并报告控制点残差。 |
| PROJ Helmert: https://proj.org/en/stable/operations/transformations/helmert.html | Helmert 变换是坐标基准转换基础模型。 | 内置轻量 2D 相似变换；正式 CRS 管线后续接 PROJ。 |
| LandXML parser: https://github.com/mac999/landxml_parser | LandXML 可表达线路中线、桩号和 profile 数据。 | 线路工具优先支持 station/offset、平纵线形和控制点复核。 |

## 已落地

1. 工作台工具范围已收敛为坐标、轨道、轨道控制、监测、线路、放样和施工控制。
2. 距离方位、角度换算、坐标转换、线路放样、线路里程偏距、轨道精调、盾构姿态、CPIII、控制网和监测类工具已有可运行计算链。
3. 控制网已支持二维边角网、一维水准网、三维 GNSS 基线、方向组、自由网内约束、稳定点约束转换、粗差诊断和二次复核成果。
4. 坐标转换已支持已知参数批量转换、同名控制点解算和 CSV/JSON 输入。
5. 线路里程偏距已支持 CSV、GeoJSON、LandXML 和 DXF 输入，并输出里程、偏距和高程偏差复核成果。
6. CPIII、线路测量表和测量基础计算表支持设计/实测坐标、毫米级较差列、dE/dN/dH 东北高差列、水准高差列、角度 DMS 列、单位后缀、分号 CSV，以及全角数字、全角/Unicode 正负号。
7. 监测和变形趋势工具已覆盖变形、水位、测斜、轴力和监测 CSV 汇总，支持表内累计/速率阈值列、宽表按测点匹配阈值列、用户输入阈值参数、中文/斜杠日期、全角中文日期、Excel 序列号日期、单位后缀、分号 CSV、全角数字、全角/Unicode 正负号、宽表/长表等常见表格，并按标准化日期计算阶段速率；水位、测斜和轴力序列也按同一日期标准化逻辑计算阶段速率。
8. 格式解析器已支持 Leica GSI、DAT/坐标表，可识别外业表中的引号字段、千分位、单位后缀和全角/Unicode 正负号，并可把解析结果转为距离计算、角度换算和坐标转换输入。
9. 图表、Excel 和报告工具已能消费各计算工具的 `export_rows`，形成趋势图、分表工作簿，以及 Word/Markdown 结构化报告。

## 近期开发顺序

1. 继续补齐计算工具的数据格式：优先处理现场常见宽表、中文表头、单位列、空行、缺测值和多测点批量输入。
2. 强化核心算法输出：每个工具都要给出计算公式对应字段、限差判断、失败原因和可追溯明细行。
3. 打通解析到计算：`survey_format_parser` 输出应能直接作为距离、角度、坐标转换、控制网和监测工具输入。
4. 打通计算到成果：所有工具统一输出 `summary` 与 `export_rows`，供图表、Excel 和报告复用。
5. 补现场样例测试：每个工具至少保留一份中文 CSV 样例、一份 JSON 样例和一份异常数据样例。

## 暂不开发

以下内容暂时不作为 Railwise 工程分析工作台目标：

1. 偏离轨道、交通、铁路、测量和监测主线的非轨道交通测量监测类计算。
2. 偏离计算复核目标的复杂交付流转和审批闭环。
3. 面向大型 GIS/点云生产的完整平台能力。相关能力只保留为后续 PROJ/GDAL/PDAL 外部引擎接入方向，不影响当前计算工具交付。
